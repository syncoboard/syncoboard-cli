Prompt for Backend Agent:

"Please implement the WebRTC Voice Call signaling endpoints for the syncoboard API. The CLI application expects the following endpoints to be available for a given board:

1. **POST /boards/:boardId/voice/join** - Registers a peer in the call.
   - Body: `{"peerId": "string"}`
   - Action: Upserts the current user in `BoardMember` (or similar active session table) and stores their WebRTC `voicePeerId` and a `lastVoicePing` timestamp.

2. **POST /boards/:boardId/voice/leave** - Removes a peer from the call.
   - Action: Clears the `voicePeerId` for the current user.

3. **GET /boards/:boardId/voice/peers** - Retrieves active peers.
   - Response: `{"peers": ["peer1", "peer2"]}`
   - Action: Returns a list of `voicePeerId` strings for all users currently in the voice call (excluding the requester) whose `lastVoicePing` is recent (e.g. within 30 seconds).

4. **POST /boards/:boardId/voice/signal/:toPeerId** - Sends a WebRTC signal.
   - Body: `{"type": "offer" | "answer" | "candidate", "data": "json_string"}`
   - Action: Saves the signaling payload to a database table/cache queue so the target peer can retrieve it. Includes the sender's `peerId`.

5. **GET /boards/:boardId/voice/signals** - Retrieves pending WebRTC signals.
   - Response: `{"signals": [{"peerId": "sender_id", "signal": {"type": "offer", "data": "..."}}]}`
   - Action: Returns and deletes pending signals targeted to the requesting user's `voicePeerId`."
